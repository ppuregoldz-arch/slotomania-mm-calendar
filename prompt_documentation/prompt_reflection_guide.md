# Prompt Reflection Guide (SM)

## Record for each analysis
- Original prompt and rubric scores
- Improvements suggested/applied
- Final prompt
- Business value and next steps

## Prompt Reflection Process

### Before Analysis
1. **Assess Original Prompt**
   - Clarity (1-5)
   - Scope (1-5)
   - Complexity (1-5)
   - Actionability (1-5)

2. **Identify Improvement Areas**
   - Missing metrics
   - Missing timeframe
   - Missing segments
   - Missing business objective
   - Currency confusion
   - Aggregation issues

3. **Suggest Improvements**
   - Specific recommendations
   - Examples of improved prompts
   - Business value explanation

### During Analysis
1. **Document Prompt Evolution**
   - Original prompt
   - Improved prompt
   - Changes made
   - Rationale for changes

2. **Track Learning**
   - What worked well
   - What could be improved
   - Patterns identified

### After Analysis
1. **Reflect on Results**
   - Did the prompt lead to valuable insights?
   - Were the results actionable?
   - What would improve the prompt further?

2. **Document Best Practices**
   - Effective prompt patterns
   - Common improvement areas
   - SM-specific considerations

## SM-Specific Considerations

### Revenue Analysis Prompts
- Always specify: Real money revenue vs virtual currency redemption
- Always specify: Use daily Net revenue (approved revenue)
- Always specify: Product group breakdown if needed

### User Engagement Prompts
- Always specify: CZ deluxe segments or VIP tiers
- Always specify: Paying vs non-paying users if relevant
- Always specify: Timeframe with date exclusion

### Economy Analysis Prompts
- Always specify: Balance distribution percentiles
- Always specify: Consumption rate calculation method
- Always specify: Currency type (coins, gems, slotobucks)

### Product Analysis Prompts
- Always specify: Product group vs SKU level
- Always specify: Value-for-money metrics if needed
- Always specify: Real money vs virtual currency products

This guide helps track prompt evolution and improve prompt quality for SM data analysis.
