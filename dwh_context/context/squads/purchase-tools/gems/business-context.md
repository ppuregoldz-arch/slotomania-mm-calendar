# GEMS - Business Context

**Feature**: GEMS 
**Purpose**: Premium in-app currency and monetization infrastructure
**Category**: Core Economy & Purchase Tools Integration

## Feature Overview

### Core Function
- **Premium Currency**: Primary in-app purchase currency for Slotomania
- **Multi-Sink Usage**: Spendable across various game features and mechanics
- **Purchasable Currency**: Direct revenue generation through gem purchases
- **Lobby Presence**: Permanent static icon in lobby for easy access

### Economic Role
- **Bridge Currency**: Connects real money to in-game value
- **Flexibility Currency**: Allows players to enhance multiple game experiences
- **Retention Tool**: Provides non-coin progression and purchase options
- **Conversion Catalyst**: Enables micro-transactions across feature ecosystem

## Business Goals

### Primary Objectives
- **Revenue Diversification**: Multi-channel monetization beyond coin packages
- **User Choice**: Provide alternative to coin-focused purchases
- **Feature Enhancement**: Enable premium experiences across game features
- **Spending Flexibility**: Allow targeted spending on specific needs/desires

### Strategic Integration
- **Cross-Feature Utility**: Usable in Daily Dash, Seasonal features, Premium upgrades
- **Retention Driver**: Non-wager currency for progression and enhancement
- **Monetization Enabler**: Powers premium tiers and instant access options
- **Value for Money (VFM)**: Scales value with player tier and level

## Key Performance Indicators

### Revenue Metrics  
- **Gem Purchase Revenue**: Direct gem package sales
- **Gem Spend Revenue**: Revenue attributed to gem-enabled purchases
- **Gem Conversion Rate**: Free-to-paid gem user conversion
- **Revenue per Gem User**: Average spending by gem-purchasing users

### Usage Metrics
- **Gem Acquisition**: Purchase vs earned gem sources
- **Gem Spending**: Distribution across different sinks and features
- **Gem Retention**: Gem balance management and hoarding patterns
- **Cross-Feature Usage**: Gem spending across different game areas

### Business Performance
- **Market Share**: Gem revenue vs total monetization
- **User Penetration**: % of users who purchase gems
- **Repeat Purchase**: Gem package repurchase behavior
- **Feature Enablement**: How gems unlock premium experiences

## Technical Architecture

### Purchase Infrastructure
- **Payment Page (PP)**: Primary gem purchase interface
- **Product Catalog**: Various gem package denominations and values
- **SKU Integration**: Standard (sku_id, transaction_source_type_id) mapping
- **Platform Support**: Cross-platform gem purchase and sync

### Spend Integration
- **Multi-Sink Architecture**: Gem spending across diverse game features
- **Balance Management**: Gem inventory and transaction tracking
- **Value Calculation**: Dynamic gem-to-value conversion by context
- **Cross-Feature API**: Standardized gem spending interface

## Gem Economy & Sinks

### Primary Spend Categories
- **Daily Dash**: Super Dash missions, skip options, premium rewards
- **Seasonal Features**: Extra picks, instant access, premium rewards
- **Premium Upgrades**: Golden Spin, Dice Deluxe, LBP Premium access
- **Instant Gratification**: Skip timers, instant opens (Mega Pods)
- **Enhancement Tools**: Boosters, multipliers, premium experiences

### Acquisition Sources
- **Direct Purchase**: Gem packages through Payment Page
- **Earned Rewards**: Daily Dash, achievements, promotional rewards
- **Premium Benefits**: VIP platform bonuses and enhanced earning rates
- **Promotional Grants**: Event rewards and retention incentives
