## Confirmed learnings

- **Parent and subitems:** Non-Reuse parent = concise Creative Label + Change summary. Actionable subitems use one vertical HTML table; no prose outside it.
- **References:** `Reference` is the matching embedded preview only. `Reference Link` is the CRM3 folder. Designer-delivered attachments outrank stale path prose.
- **Generic work:** Do not use a holiday/themed asset for a generic brief. Use themed references only when MM explicitly requests that same theme.
- **Gems Coupon:** `Inapp` only; no `store denom`. Use the designer-uploaded Inapp PNG from Monetization-Art pulse `11415353948`; link `Q:\Slotomania\CRM3\Features\Gems\2026\2026_03_14_Gem_Coupon\Inapp`. Themed art requires the explicit MM token `ALLOW_THEMED_GEMS_COUPON`.
- **MGAP basic UI:** Exactly one subitem named `MGAP UI`; never `Full theme UI` unless MM explicitly asks for full-theme execution.
- **Prize Mania:** Pricing is configuration, not Creative input. The visible prizes in MM Description are the Hook and must appear in the Creative delta.
- **Pricing generally:** Strip `H/M/L/Max` pricing from Creative titles, Change, What to change, and Hook.
- **New promo:** Use the New promo skeleton with CTA; never ship generic Task/Keep pointer rows.
- **Battlesheep / Blast:** Default scope is Main Inapp + Banner + Winners Inapp. Add challenge assets and SKU only when MM Description explicitly defines a season challenge.
- **Decoy:** Do not create Monetization-Art briefs for Decoy Bonanza / multi-denom decoy rows.
- **Rolling + MGAP:** Default Rolling scope is Background + Banner; add only the named `MGAP denom` work when the MM row includes an MGAP ladder. Do not create generic buy/free denom briefs.
- **Hook placement:** Hook rows belong only on Banner and Inapp assets.
- **Inapp scope:** Do not add Winners Inapp to Daily Deal store offers or Spinner Clash. Follow the product playbook for product-specific exceptions.
- **Status MM:** The agent uses `Waiting for MM` after writing a complete brief. The MM owner moves it to `Ready for Brief`.
- **People mapping:** First Creative Traffic owner = Artist; second = Copywriter. Never put both only under Artist.
- **Reuse:** One consolidated `REUSE - No Creative Action` item per day, with exact source evidence and no asset subitems.
